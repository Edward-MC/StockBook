---

## 1. 定位

个人、单用户、本地优先的 Web 应用,围绕一套**具体的投资策略**组织记录与追踪:

> **大类资产配置 + 区间再平衡**(target allocation + tolerance-band rebalancing)

- 标的范围:**A 股**(含场内基金/ETF),人民币计价。
- 核心闭环:**记账(在标的层面记买卖)→ 追踪(自下而上汇总到大类、对比目标)→ 再平衡建议(漂出区间就提示加/减仓及金额)**。
- 区别于通用基金 App:一切围绕「我自己设定的目标配置和容忍区间」来组织,而不是单纯展示净值。

### 设计哲学:为「现在」做,为「以后」留接口

用户当前对策略的认知仍在成形,因此 v1 做**可配置的容器**而非硬编码规则;同时数据模型从一开始就「策略感知」「市场无关」「价格可刷新」,使后续扩展是顺势加列/加行,而非推倒重构。

## 2. 范围(v1)

| v1 **做** | 明确**不做**(留给后续版本) |

## 3. 领域模型(SQLite + SQLAlchemy)

两层结构:**标的(Security)归入大类(AssetClass)**。持仓与成本由交易**推导**,不单独存表。

### 3.1 实体

- **Strategy 策略容器**
  - `id`, `name`(v1 仅一条,如「我的核心配置」)
  - 大类目标/区间挂在其名下 → 以后加第二套策略是加行,不改表结构。

- **AssetClass 大类**(可增删改)
  - `id`, `strategy_id`(外键)
  - `name`(如 沪深300 / 中证500 / 红利 / 行业 / 债券)
  - `target_weight`(目标权重 %,自上而下用拨杆设)
  - `band_low`, `band_high`(容忍区间下/上限 %)
  - `color`(配色,对应原型 --c1..--c5)
  - `sort_order`(展示顺序)

- **Security 标的**(可在某大类下手动添加)
  - `id`, `asset_class_id`(外键)
  - `code`(如 `510300`)、`name`
  - `market`(默认 `CN`,为美股预留)

- **Transaction 交易**(记账原始数据)
  - `id`, `security_id`(外键)
  - `date`、`action`(`buy` / `sell`)、`shares`(股数)、`price`(成交价)
  - 持仓股数 = Σ(买入股数) − Σ(卖出股数);成本由交易推导。

- **PriceQuote 现价**(每标的最新价)
  - `security_id`(外键)、`price`、`updated_at`
  - `source`(默认 `manual`;以后自动源填 `auto`)
  - v1 手动维护;每个标的保留「最新一条」即可。

### 3.2 推导值(不入库,实时计算)

- 标的持仓股数 → 标的市值 = 持仓股数 × 现价
- 大类市值 = Σ(该大类下各标的市值)
- 总资产 = Σ(各大类市值)
- **大类当前占比** = 大类市值 ÷ 总资产
- **标的在大类内占比** = 标的市值 ÷ 大类市值
- **标的在总资产占比** = 标的市值 ÷ 总资产
- **偏离** = 大类当前占比 − 目标占比
- **状态**:当前占比 < band_low → 需加仓;> band_high → 需减仓;区间内 → 合规
- **再平衡金额** = (目标占比 − 当前占比) ÷ 100 × 总资产(正=加仓,负=减仓)

## 4. 「配置策略」交互:未分配池模型

> 替换原型里「其余大类自动按比例吸收差额」的逻辑,改为更直观可控的**未分配池(额外百分比)**。

- 定义 **未分配 % = 100 − Σ(所有大类目标权重)**。
- 拖动某大类拨杆**调低** → 释放的百分比**回到未分配池**(未分配 ↑)。
- 拖动某大类拨杆**调高** → 从未分配池**取用**(未分配 ↓);**池子不能为负**(取完即不能再加)。
- 拖动**互不牵连**:动一根拨杆不会自动改动其他大类。
- **新增一个大类**:默认目标 0%,随后滑上去从未分配池取额度。
- **删除一个大类**:其目标权重退回未分配池。
- **配置完成的标志 = 未分配池归 0**(即 100% 全部分配)。
- 顶部用醒目 chip 显示当前未分配值:**= 0 显示绿色「已配平」**,≠ 0 显示提示色。

**两个占比并列展示**(核心对比):
- **目标占比**:拨杆设的值 → 目标堆叠条 + 轨道上黑色竖线(标「目标」)。
- **当前占比**:标的市值自下而上汇总 → 实际堆叠条 + 轨道上圆点。
- 偏离 = 当前 − 目标,直接显示 `+3.0%` / `−6.0%`;圆点漂出浅绿容忍区间即标「需加/减仓」。

**锁定按钮:v1 去掉**(在未分配池模型下失去原本「防止被自动分摊连带改」的作用)。见 §10 未决问题——若日后想要「防误改锁」可再加。

## 5. 界面

单页为主,套用用户提供的视觉风格:暖米色纸质感背景 + 颗粒噪点、`Fraunces`/`Noto Serif SC` 标题、`Noto Sans SC` 正文、`DM Mono` 数字、那套配色与升起动效。

### 5.1 仪表盘(主页,= 用户原型,数据接真实后端)
- 顶部:总资产、偏离大类计数、估值日期。
- **目标配置**:目标堆叠条 + 实际堆叠条 + 图例;每个大类一根拨杆(未分配池模型);未分配 chip;「重置为默认」「新增大类」「编辑大类」入口。
- **持仓与偏离**:每个大类一条轨道(容忍区间浅绿带、目标竖线、当前圆点、偏离数值、状态 chip);展开可见**该大类下的标的列表**,每个标的显示:名称/代码、市值、**大类内占比**、**总资产占比**。
- **再平衡建议**:把漂出区间的大类拉回目标所需的加/减仓金额列表;全部合规时显示「无需调仓」。

### 5.2 录入页(v1 朴素表单即可)
- 新增/编辑大类(名称、目标、区间、配色)。
- 在某大类下新增标的(代码、名称)。
- 记一笔交易(标的、日期、买/卖、股数、成交价)。
- 更新某标的现价。

### 5.3 只读分享模式
- 一个开关(URL 参数或配置项):开启后**隐藏所有录入入口、拨杆禁用为只读**,可选「只显示百分比、隐藏金额」。
- 即「把我的配置/持仓/再平衡只读地给别人看」。

## 6. 架构与技术栈

**单进程:FastAPI(后端 + Jinja2 模板) + SQLite**,零 Node 构建——因为原型本身是原生 HTML/CSS/JS,直接复用。

- 后端:FastAPI + SQLAlchemy + SQLite,Pydantic 校验。
- 前端:Jinja2 渲染页面外壳 + 用户那套 CSS;原生 JS 通过 `fetch` 调 JSON API 读写数据。拨杆/未分配池/再平衡均为客户端计算,保存时写库。
- **JSON API**(示意):
  - `GET /api/dashboard` — 一次性返回页面所需:各大类(目标/区间/当前占比/市值/配色)+ 其下标的(市值/两个占比)+ 再平衡建议 + 总资产/估值日期。
  - 大类 CRUD:`POST/PUT/DELETE /api/asset-classes`
  - 标的 CRUD:`POST/PUT/DELETE /api/securities`
  - 交易:`POST/DELETE /api/transactions`
  - 现价:`PUT /api/securities/{id}/price`
  - 目标配置保存:`PUT /api/strategy/targets`

## 7. 错误处理与校验

- **目标合计**:保存目标配置时,后端校验 未分配 = 0(Σ目标 = 100);≠ 0 时返回提示,前端 chip 已实时反映。
- **卖出校验**:某标的累计卖出股数不得超过累计买入(持仓不能为负)。
- **数值校验**:股数、成交价、现价均须为正;区间须 `band_low ≤ target ≤ band_high` 且 `0 ≤ band_low < band_high ≤ 100`(软提示,不强制阻断)。
- **缺数据兜底**:某标的未填现价 → 标注「待估值」,**不纳入市值汇总**以免污染占比;界面提示哪些标的待估值(对应原型底部「净值实时接口待接入」状态)。

- 缺现价兜底、卖出超持仓等边界用例。

## 11. 已确认的关键决策(对话纪要)

1. 围绕个人投资策略,而非通用基金 App。
2. 策略具体为「大类配置 + 区间再平衡」。
3. v1 单一策略,但数据模型策略感知,为多策略预留。
4. A 股优先,美股后续(market-agnostic 模型)。
5. v1 手动录入交易与现价,价格字段「可刷新」,自动源后续接。
6. 两层持仓:标的归入大类,标的层面记账、自动汇总到大类。
7. Web 界面,采用用户提供的「衡」原型视觉风格。
8. 单用户、本地优先;只读分享「我的」,可打包给别人跑;无账号系统。
9. 拨杆采用「未分配池」模型;大类两个占比(目标 vs 当前)并列;标的显示大类内占比与总资产占比;锁定 v1 去掉。
